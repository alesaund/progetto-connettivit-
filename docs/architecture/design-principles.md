# Design Principles

## Missione
Costruire una rete stabile, osservabile e facilmente manutenibile.

## Principi
1. Simplicity over complexity.
2. Documentation as Code.
3. Infrastructure as Documentation.
4. Reboot preferito al recovery incompleto quando garantisce maggiore affidabilità.
5. Ogni modifica deve essere riproducibile.
6. Nessuna configurazione senza motivazione documentata (ADR).
7. Preferire soluzioni stabili a soluzioni teoricamente più performanti.
